Harmony Haven Stable v2.1.0 — Category Intelligence
====================================================

ฐานที่ใช้พัฒนา
- ต่อจาก Harmony Haven Stable v2.0.1 — Project Expense UX
- ไม่รื้อระบบ Login, Cloud State, Calendar, Project, Notification หรือข้อมูลเดิม
- ไม่ต้องรัน SQL ใหม่ หากเคยรัน 003_cloud_household_state.sql และ 004_calendar_notifications.sql แล้ว

สิ่งที่เพิ่มในเวอร์ชันนี้

1) หมวดหมู่แบบ 3 ระดับ
- หมวดหลัก
- หมวดย่อย
- รายละเอียดระดับ 3

ตัวอย่าง
- รถยนต์ › ซ่อมบำรุง › ยาง
- อาหาร › ร้านอาหาร › ปิ้งย่าง
- สุขภาพ › ยาและเวชภัณฑ์ › ยา

2) หมวดเริ่มต้นแบบละเอียด
- อาหาร
- เครื่องดื่ม
- ของใช้ในบ้าน
- ค่าน้ำค่าไฟ
- เดินทาง
- รถยนต์
- ที่อยู่อาศัย
- สุขภาพ
- ท่องเที่ยว
- ช้อปปิ้ง
- บันเทิง
- การศึกษา
- ธุรกิจและงาน
- รายได้
- อื่น ๆ

3) รองรับหมวดละเอียดในทุกจุดสำคัญ
- บันทึกรายรับ–รายจ่ายทั่วไป
- ค่าใช้จ่ายในโปรเจกต์
- งบประมาณรายเดือน
- รายการประจำ
- ค่าใช้จ่าย/บิลในปฏิทิน

4) Tag
- เพิ่ม Tag ได้หลายคำในหนึ่งรายการ
- ตัวอย่าง: ทริปเชียงใหม่, ลูกค้า, รายเดือน
- ค้นหา Tag จากหน้ารายการย้อนหลังได้
- CSV จะส่งออกทั้งหมวดหลัก หมวดแบบละเอียด และ Tag

5) Smart Category
- ระบบจำหมวดและ Tag ตามชื่อร้านค้าหรือรายการที่เคยบันทึก
- ครั้งถัดไปเมื่อกรอกชื่อเดิม ระบบจะเลือกหมวดเดิมให้อัตโนมัติ
- มีคำแนะนำเบื้องต้นสำหรับปั๊มน้ำมัน คาเฟ่ เดลิเวอรี รถรับจ้าง ซูเปอร์มาร์เก็ต สตรีมมิ่ง โรงพยาบาล และโรงแรม
- ผู้ใช้ยังเปลี่ยนหมวดเองได้เสมอ

6) จัดการหมวดจากหน้าตั้งค่า
- เพิ่ม/ลบหมวดหลัก
- เพิ่ม/ลบหมวดย่อย
- เพิ่ม/ลบรายละเอียดระดับ 3
- รายการเก่าจะไม่ถูกลบ แม้ลบหมวดออกจากรายการตัวเลือก
- จัดการคลัง Tag
- ดูและล้างความจำร้านค้าได้

7) ความเข้ากันได้กับข้อมูลเดิม
- รายการเก่าที่มีเฉพาะ category ยังแสดงผลและคำนวณได้
- งบเก่าและรายการประจำเก่ายังใช้งานได้
- Cloud State ปรับเป็น schemaVersion 3 ภายใน JSON เดิม จึงไม่ต้องแก้ตาราง Supabase

8) Stability Fix ที่รวมมาด้วย
- เพิ่ม UUID fallback กรณีเบราว์เซอร์ไม่มี crypto.randomUUID
- เพิ่มฟังก์ชันและ Event ของเรทสกุลเงินในฟอร์มรายการทั่วไปที่ขาดในฐานเดิม
- ตรวจสอบ ID ใน HTML กับ JavaScript ไม่พบ Reference ที่หาย

วิธีอัปเดต
1. สำรองหรือ Commit โปรเจกต์ปัจจุบัน
2. อัปไฟล์ทั้งหมดใน ZIP ทับโปรเจกต์เดิม
3. ไม่ต้องรัน SQL เพิ่ม หาก 003 และ 004 เคยรันแล้ว
4. Deploy บน Vercel โดย Clear Build Cache
5. เปิดเว็บแล้วกด Ctrl+F5
6. ถ้าใช้ PWA ให้ปิดแอปจาก Recent Apps แล้วเปิดใหม่

หมายเหตุ
- งบประมาณสามารถตั้งระดับหมวดหลัก หมวดย่อย หรือระดับ 3 ได้
- งบหมวดหลักจะรวมค่าใช้จ่ายของหมวดย่อยทั้งหมด
- ระบบยังสรุปกราฟหน้า Dashboard ตามหมวดหลัก เพื่อให้อ่านง่าย
