Harmony Haven Stable v2.0.0

ฐานโค้ด
- สร้างจาก harmony-haven-stable-history-date-range
- คงข้อมูล Cloud, Calendar, Tasks, Reminders และ Notifications เดิม
- ไม่เพิ่ม Activity Center

สาเหตุที่เวอร์ชันก่อนหน้าเอ๋อ
1. หน้า “รายการย้อนหลัง” ลบ input id="filterMonth" ออกแล้ว
   แต่ JavaScript ยังรัน $('filterMonth').value = ...
   ทำให้ JavaScript หยุดตั้งแต่เปิด app.html
2. Calendar เรียก escapeHtml() แต่ในฐาน Stable ไม่มีฟังก์ชันนี้
3. app.html มีหน้าเลือกผู้ใช้ซ้ำกับหน้า PIN หลัก และมี Redirect ทันทีเมื่อหา activeUser ไม่เจอ
   ทำให้การโหลด Cloud/Session ที่ช้ากว่าปกติมีโอกาสชนกัน

สิ่งที่แก้ใน v2.0.0
- แก้ filterMonth null error
- เพิ่ม escapeHtml แบบปลอดภัย
- ซ่อนหน้าเลือกผู้ใช้ภายใน app.html และใช้หน้า PIN หลักเป็นจุดเข้าเพียงจุดเดียว
- เพิ่ม Boot Controller: ตรวจ Session/Profile/Cloud ก่อนเปิดหน้าใช้งาน
- Cloud ชั่วคราวขัดข้องจะค้างที่หน้าลองใหม่ ไม่เด้งวนกลับ Login
- API timeout 15 วินาที พร้อมปุ่มลองใหม่
- Save Cloud แบบ debounce 800 ms และป้องกันการบันทึกซ้อน
- แยก JavaScript ออกจาก HTML เป็น public/assets/harmony-app-v2.js
- renderAll จับ Error แยกแต่ละโมดูล ไม่ให้หน้าเดียวทำทั้งระบบค้าง
- รองรับรายการเก่าที่ไม่มี splits
- รายการย้อนหลังเลือก วันนี้ / 7 วัน / 30 วัน / เดือนนี้ / เดือนก่อน / ปีนี้ / กำหนดช่วงเอง
- ตรวจวันที่เริ่มต้นไม่ให้เกินวันที่สิ้นสุด
- เรียงรายการใหม่ไปเก่า
- ลดการ render ซ้ำระหว่างพิมพ์ตัวกรอง

วิธีติดตั้ง
1. สำรองโปรเจกต์ปัจจุบันหรือสร้าง Git commit ก่อน
2. คัดลอกไฟล์ทั้งหมดใน ZIP ทับโปรเจกต์
3. ไม่ต้องรัน SQL เพิ่ม ถ้าเคยรัน 003 และ 004 แล้ว
4. Deploy บน Vercel โดยเลือก Clear Build Cache
5. เปิดแอปด้วยเบราว์เซอร์ กด Ctrl+F5 หนึ่งครั้ง
6. สำหรับ PWA ให้ปิดแอปจาก Recent Apps แล้วเปิดใหม่

ไฟล์หลักที่เปลี่ยน
- public/app.html
- public/assets/harmony-app-v2.js
- app/api/cloud/state/route.js (รวมไว้เพื่อให้ชุดติดตั้งครบ)
